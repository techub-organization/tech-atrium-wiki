# Architecture Decision Records (ADR)

## 概要

`docs/adr/` は、tech-atrium-wiki における取り消しにくいアーキテクチャ上の決定を記録するディレクトリである。

ADR により、なぜその選択をしたのか・何を却下したのかが将来の開発者にも伝わる状態を保つ。

## ADR が必要なタイミング

- デプロイ方式の確定（案 A / B / C）
- バックエンドサービスの追加（`stack/` 実装の本番統合）
- 認証方式の選択（Supabase Auth / Auth.js）
- DB の移行・追加
- その他、取り消しにくいアーキテクチャ上の判断

## ADR ファイルの命名

```
docs/adr/<番号>-<topic>.md
例: 0002-deploy-option.md
```

## ADR テンプレート

```markdown
# ADR-XXXX: タイトル

## ステータス

提案 / 採用 / 却下 / 廃止

## コンテキスト

なぜこの決定が必要になったか。

## 決定

何を選んだか。

## 理由

選んだ理由。

## 却下した選択肢

検討したが採用しなかった選択肢とその理由。

## 影響

この決定によって生じる制約・影響・トレードオフ。
```

# Go で作れるもの（中級者・開発者向け）

## 位置づけ

このメモは、Go の文法を少し理解した開発者が「結局 Go は何を作ると強いのか」を掴むための整理である。言語紹介ではなく、用途別の実務イメージを持つためのページとして扱う。

## 先に結論

Go は特に次のようなものと相性がよい。

- Web API
- CLI ツール
- バッチ処理
- インフラ・クラウド周辺ツール
- 軽量なマイクロサービス
- データ変換や整形の補助ツール

## 1. Web API

Go の代表的な用途の1つが Web API である。

向いている理由:

- 標準ライブラリだけでも HTTP サーバーを立てやすい
- JSON を扱いやすい
- 実行性能と保守性のバランスがよい
- コンテナ化やデプロイがしやすい

具体例:

- 記事データを返す API
- 比較データを返す API
- 認証の背後で動く内部 API
- 社内管理画面向け API

## 2. CLI ツール

Go は CLI と非常に相性がよい。

向いている理由:

- 単一バイナリで配布しやすい
- 起動が速い
- OS ごとの配布が比較的しやすい
- 設定ファイルや JSON を読むツールを書きやすい

具体例:

- データ検証コマンド
- Markdown や JSON の整形ツール
- 社内運用用コマンド
- デプロイ補助ツール

## 3. バッチ処理

定期実行や整形処理にも使いやすい。

向いている理由:

- 処理内容が明快なら実装を整理しやすい
- 並行処理を入れやすい
- 長く運用するジョブでも構成が複雑化しにくい

具体例:

- CSV / JSON の変換
- 外部 API からのデータ収集
- ログの集計
- キャッシュ更新

## 4. インフラ・クラウド周辺ツール

Go が強いと言われやすい領域。

向いている理由:

- 実行ファイルとして扱いやすい
- サーバーサイドでの長期運用に向く
- 既に Go 製ツールのエコシステムが厚い

具体例:

- Kubernetes 周辺ツール
- 監視補助ツール
- リソース管理 CLI
- 構成管理・運用補助コマンド

## 5. 軽量なマイクロサービス

Go は大規模システムの中の小さなサービスにも向く。

向いている理由:

- 起動やデプロイが軽い
- API サービスとして必要十分な性能を出しやすい
- チーム開発でコードルールを揃えやすい

具体例:

- 検索補助サービス
- 通知送信サービス
- 集計専用サービス
- 外部連携用アダプタ

## 6. データ変換や整形の補助ツール

techrium の文脈でも扱いやすい用途。

向いている理由:

- JSON / YAML / CSV の処理を書きやすい
- 小さなユーティリティとして切り出しやすい
- Python より型を持たせたい場面で使いやすい

具体例:

- 記事メタデータの検証
- 比較表データの整形
- サンプルコード一覧の生成
- API レスポンスのスキーマ確認

## Go があまり主戦場ではないもの

### 1. フロントエンドの主実装

一般的な Web フロントエンドでは TypeScript / React などが主役であり、Go を中心に据えることは少ない。

### 2. データ分析・機械学習の中心処理

可能ではあるが、エコシステムの厚さでは Python に分がある。

### 3. 高度な低レイヤー最適化が最重要の領域

Go でも可能な範囲は広いが、そこを最優先するなら Rust や C++ が候補になりやすい。

# Go と Python / Rust 比較（中級者・開発者向け）

## 位置づけ

このメモは、Go を単独で学ぶためではなく、既に他言語の感覚を持つ開発者が「なぜここで Go を選ぶのか」を比較しやすくするための wiki 的整理である。

## 先に結論

- 速く形にしたいが、Python より型と配布の安定感がほしいなら Go
- 性能やメモリ安全性を強く突き詰めたいなら Rust
- データ分析やスクリプト資産を優先するなら Python

Go は、実装速度、運用しやすさ、性能、保守性のバランスを取りやすい位置にある。

## Go と Python の比較

### Go が向く場面

- 長く運用する API を作りたい
- 型で最低限の事故を減らしたい
- 単一バイナリで配布したい
- 並行処理をある程度素直に書きたい
- 実行速度やリソース効率も意識したい

### Python が向く場面

- まず最速で試作したい
- データ分析、機械学習、スクリプト資産を活かしたい
- 学習コストをできるだけ低くしたい
- ライブラリエコシステムの広さを優先したい

### 差分の見え方

| 観点 | Go | Python |
|---|---|---|
| 型 | 静的型付け | 動的型付け |
| 実行速度 | 比較的速い | 相対的に遅め |
| 配布 | 単一バイナリで扱いやすい | 実行環境依存が出やすい |
| 文法 | 少なめで明快 | 読みやすいが自由度もある |
| 並行処理 | goroutine が使いやすい | 用途次第で選択肢が分かれる |
| 得意分野 | API、CLI、インフラツール | スクリプト、データ処理、ML |

### 判断の目安

- Python で十分だが、運用が長くなり型や配布を固めたい: Go
- Python の周辺ライブラリが主目的: Python のまま
- バックエンドに寄せて学びたい初学者: Go が選びやすい

## Go と Rust の比較

### Go が向く場面

- チーム全体で読みやすさを優先したい
- 高性能は欲しいが、実装負荷は抑えたい
- Web API、CLI、インフラツールを安定して作りたい
- メモリ安全性よりまず保守性と速度のバランスを取りたい

### Rust が向く場面

- 性能やメモリ制御をより厳密に扱いたい
- 低レイヤー寄りの処理も視野にある
- コンパイル時保証を強く活かしたい
- 学習コストが高くても安全性を取りたい

### 差分の見え方

| 観点 | Go | Rust |
|---|---|---|
| 学習コスト | 比較的低い | 高い |
| 読みやすさ | チームで揃えやすい | 慣れるまで重い |
| 性能 | 十分高い | さらに高性能を狙いやすい |
| メモリ安全性 | GC ベースで扱いやすい | 所有権で強く保証 |
| 実装速度 | 速め | 慣れるまでは遅くなりやすい |
| 得意分野 | API、CLI、運用ツール | 高性能処理、システム寄り開発 |

### 判断の目安

- まず動くものを堅実に作る: Go
- 速度、安全性、低レイヤー制御を強く取りに行く: Rust
- チームの平均習熟度を考慮するなら Go が入りやすい

## 3言語を並べたときの見取り図

| 観点 | Go | Python | Rust |
|---|---|---|---|
| 学びやすさ | 高い | 高い | 低め |
| 型安全性 | 中 | 低〜中 | 高 |
| 実装速度 | 高 | 高 | 中 |
| 実行性能 | 中〜高 | 低〜中 | 高 |
| 運用しやすさ | 高 | 中 | 中 |
| データ分析資産 | 低 | 高 | 低 |
| API との相性 | 高 | 高 | 中〜高 |
| インフラ文脈 | 高 | 中 | 中〜高 |

# Stack Map

## language

- `python`: Python の解説・データ整形・検証スクリプト・API サンプル・記事素材
- `dart`: Dart の言語特性・モバイル連携・型システムの解説サンプル
- `rust`: 高速処理の検証 PoC・CLI 補助ツール・型安全なデータ検証サンプル
- `ruby`: Ruby の記述スタイル・メタプログラミング・Web 向けサンプル
- `typescript`: TypeScript の型活用・フロント/バック共通型定義・検証サンプル
- `go`: Go の並行処理・シンプルな HTTP サーバー・CLIツール検証
- `java`: Java の設計パターン・エンタープライズ向け実装サンプル
- `php`: PHP の実装パターン・Composer エコシステム解説
- `csharp`: C# の型システム・.NET エコシステム解説サンプル
- `kotlin`: Kotlin の JVM 互換性・Android 向け知見・サンプル
- `scala`: Scala の関数型プログラミング・型システム解説
- `elixir`: Elixir の並行処理モデル・Phoenix との比較サンプル
- `clojure`: Clojure の関数型スタイル・LISP 構文解説
- `haskell`: Haskell の純粋関数型・型推論の解説
- `ocaml`: OCaml の型システム・パターンマッチ解説
- `erlang`: Erlang の分散処理・障害耐性モデル解説
- `perl`: Perl のテキスト処理・正規表現・レガシーシステム知見
- `lua`: Lua の組み込み利用・軽量スクリプト解説

## framework

- `react`: フロント統合観点の共通知見・コンポーネント設計パターン
- `flutter`: モバイル UI の実装パターン・Dart との組み合わせ解説
- `flask`: 軽量 Python Web アプリの学習・PoC サンプル
- `express`: Node.js 系の軽量 HTTP サーバー解説・ミドルウェア設計
- `nestjs`: TypeScript による構造化サーバーサイドの学習サンプル
- `fastapi`: Python 型付き API 設計の学習・PoC
- `rails`: Ruby on Rails の規約ベース開発・ActiveRecord 解説
- `laravel`: PHP MVC フレームワークの設計パターン解説
- `spring-boot`: Spring Boot のエンタープライズ設計・DI 解説
- `django`: Python フルスタック Web の ORM・認証解説
- `gin`: Go 軽量 HTTP フレームワークのサンプル・PoC
- `phoenix`: Elixir / LiveView の並行処理・リアルタイム通信解説
- `actix`: Rust 高速 Web フレームワークの学習・ベンチマーク PoC
- `ktor`: Kotlin 非同期 Web フレームワーク解説
- `micronaut`: JVM 軽量フレームワーク・GraalVM ネイティブ解説
- `symfony`: PHP フレームワークのコンポーネント設計解説
- `hono`: TypeScript 軽量エッジ対応フレームワーク解説
- `fiber`: Go 高速フレームワーク・Express ライクな API 設計解説
- `rocket`: Rust 型安全 Web フレームワーク解説
- `tide`: Rust 軽量非同期 Web フレームワーク解説
- `remix`: React ベースのフルスタック Web フレームワーク解説
- `nuxt`: Vue.js ベースの SSR フレームワーク解説・比較

## database

- `sqlite`: 組み込み・軽量 DB の特性・使い所・接続サンプル
- `postgres`: PostgreSQL 設計・スキーマ知見・運用ノウハウ・接続サンプル
- `mysql`: MySQL リレーショナル DB の設計・チューニング知見
- `mariadb`: MariaDB の MySQL との互換性・差異解説
- `mongodb`: ドキュメント DB の設計パターン・集計パイプライン解説
- `redis`: キャッシュ・Pub/Sub・セッション管理の使い方解説
- `cockroachdb`: 分散 PostgreSQL 互換 DB の特性・スケールアウト解説
